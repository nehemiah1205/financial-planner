# 家庭財務健康體檢與目標模擬系統（網頁互動版）

由「財務試算表新版_2026_.xlsx」轉製而成的 Streamlit 互動網頁應用。

## 如何在本機執行

```bash
pip install -r requirements.txt
streamlit run app.py
```

執行後瀏覽器會自動開啟 `http://localhost:8501`，左側選單可切換各模組。
（若要放到網路上給客戶使用，可部署到 Streamlit Community Cloud、或公司內部伺服器。）

## 新增：資料存檔與讀取（解決關閉分頁資料消失的問題）

左側選單最上方「💾 資料存檔與讀取」頁面：
- **下載備份**：把目前所有已輸入的資料（家庭成員、財務目標、Step3 收支資產負債表含調整後版本、退休參數，以及各專題頁面已輸入的數字、顧問建議文字等）打包成一個 `.json` 檔下載，檔名自動帶入客戶姓名與日期
- **上傳還原**：下次要繼續作業時，上傳同一份 `.json` 檔，系統會自動還原所有頁面的資料，不用重新輸入
- 採用「自動掃描」方式打包資料，日後不管加多少新頁面或新欄位，都不需要再回頭修改這個存檔功能

⚠️ 提醒：Streamlit 網頁本身**不會**幫您保存資料（關閉分頁或重新整理就會清空），這是所有 Streamlit 應用共通的限制，所以才需要這個「手動下載/上傳備份」的機制。若之後要做到「自動存檔、不用手動下載」，需要接資料庫（例如 Google Sheets、Firebase），這部分我可以之後再協助評估。

## 新增：客戶總覽報告

左側選單最下方「📋 客戶總覽報告」是專為分享給客戶設計的重點彙整頁：
- 家庭成員、財務現況（收入/支出/結餘/淨資產）
- 財務健康四大指標（儲蓄率、負債比、生活週轉金、財務自由度）與狀態燈號
- 各專題（退休/教育金/保障/購屋/購車）的資金缺口與建議每月投入金額彙總表（購屋、購車可切換要顯示 A/B/C 哪個計畫）
- 顧問建議欄位（可自行編輯文字）
- 頁面已針對「瀏覽器列印」優化：按 `Ctrl+P`（Mac 為 `Cmd+P`）→選「另存為PDF」，會自動隱藏側邊選單，適合直接印出或存成 PDF 傳給客戶
- 也提供「下載文字摘要 (.txt)」按鈕，方便貼到 Email / LINE

💡 建議操作流程：先在 Step1~4 與各專題完成試算，最後再切到「客戶總覽報告」查看彙整結果。

## 修正：輸入數字後「打完就跑掉」的問題

**原因**：Step 3 收支表、Step 1 退休參數表等可編輯表格，原本的程式邏輯是「每次畫面重新整理，都用另一份暫存資料重新蓋出一個新表格，再丟給編輯元件」。
Streamlit 的可編輯表格如果每次收到的來源資料跟前一次不完全一樣（即使數值相同、只是物件不同），就會誤判成「資料被別處改了」而重置使用者剛輸入的內容 —— 這就是您打完數字後又跳回 0 的原因。

**修正方式**：改成「表格只存一份，且永遠直接沿用編輯元件自己回傳的最新結果」，不再另外重建一份新表格。這是 Streamlit 官方建議的正確寫法，修正後數字會確實留住，不會被重置。

已修正的表格：Step 3 收支/資產/負債各表、Step 1 退休參數表、家庭成員表（並把「年齡」欄位改為獨立顯示，避免影響可編輯表格本身）。

## 這次優化了什麼

1. **資料一次輸入、全站同步**
   - Step 3 的收支與資產負債資料，會自動帶入 Step 4 財務健康看板（原本 Excel／舊版程式是兩邊各自輸入，容易兜不起來）。
   - Step 1 的退休經濟假設參數（通膨率、投報率…）會自動帶入「專題 A：退休規劃」，不用重複填兩次。
   - 家庭成員資料（姓名、性別、生日）會自動連動到退休、教育金、保險等所有試算專題。

2. **對齊 Excel 原本的完整結構**
   - Step 3 收支表拆成「一般 / 理財 / 保障型 / 其他」四張子表，並新增「調整後建議」分頁可與現況並排比較。
   - 教育金試算支援最多 4 位子女，每位子女可分別設定大學／碩士／博士三階段，並可直接選擇 Excel 參數表中的學費預設值（國內外公立/私立大學、研究所、博士班）。
   - 購屋、購車試算改為 A / B / C 三計畫並排比較，架構對應原 Excel 的「購屋試算」「購車試算」工作表。
   - 保險缺口分析新增住院日額「本人／配偶／子女1／子女2」多欄位比較表。

3. **修正原本 Python 版的問題**
   - 原程式 `step1_basic_info()` 被定義兩次（第一版完全是死碼），已整合成單一版本。
   - 原程式 Step 4 與退休模組在側邊欄用「另一組輸入框」模擬 Step 1/3 的資料，會跟真正輸入的資料脫勾；現在改為直接讀取同一份 `session_state`。
